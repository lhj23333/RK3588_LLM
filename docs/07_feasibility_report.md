# RK3588 (8GB) 模型可行性分析报告

**测试范围说明**：8B 模型（Qwen3-8B、InternVL3.5-8B）在 8GB 板子上会 OOM，本项目**不纳入测试**；下表仍保留其可行性分类供参考。

## 硬件约束

| 项目 | 规格 |
| :--- | :--- |
| SoC | Rockchip RK3588 |
| DRAM | 8GB (MemTotal: 8110608 kB) |
| NPU | 6 TOPS (3 cores x 2 TOPS) |
| OS | Ubuntu (Linux 5.10.160, aarch64) |
| RKLLM Runtime | v1.2.3 |
| RKNPU Driver | v0.9.8 |

系统本身 (内核 + 桌面 + 后台服务) 常驻约 2-2.5GB，因此可用于模型加载的实际内存约为 **5.5-6GB**。

---

## 1. 老板要求的模型清单 vs 实际可行性

### 1.1 InternVL3.5 系列

| 模型 | 参数量 | RKLLM 支持 | 量化方式 | 预估 RAM (MB) | 可行性 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| InternVL3.5-1B | 1B | v1.2.3 支持 | W8A8 | ~1,900 | **可以跑** | 最轻量，推荐首选验证 |
| InternVL3.5-2B | 2B | v1.2.3 支持 | W8A8 | ~3,000 | **可以跑** | 内存充裕 |
| InternVL3.5-4B | 4B | v1.2.3 支持 | W8A8 | ~5,400 | **可以跑** | 剩余空间较小但稳定 |
| InternVL3.5-8B | 8B | v1.2.3 支持 | W8A8 | ~8,800 | **不测试/OOM** | 8GB 会 OOM，已排除 |
| InternVL3.5-14B | 14B | 未验证 | - | >14,000 | **不可行** | 远超 8GB 物理内存 |
| InternVL3.5-30B-A3B | 30B (MoE) | 未验证 | - | >16,000 | **不可行** | MoE 结构需加载全部 30B 权重到内存 |

### 1.2 Qwen3 纯文本系列

> **注意**：老板列表中的 "Qwen3-1B" 实际不存在。Qwen3 系列的密集模型尺寸为: 0.6B, 1.7B, 4B, 8B, 14B, 32B。最接近 1B 的是 0.6B 和 1.7B。

| 模型 | 参数量 | RKLLM 支持 | 量化方式 | 预估 RAM (MB) | 可行性 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| Qwen3-0.6B | 0.6B | v1.2.1+ 支持 | W8A8 | ~774 | **可以跑** | 最轻量文本模型 |
| Qwen3-1.7B (老板说的"1B"?) | 1.7B | v1.2.1+ 支持 | W8A8 | ~1,700 | **可以跑** | 需确认老板意图 |
| Qwen3-4B | 4B | v1.2.1+ 支持 | W8A8 | ~4,300 | **可以跑** | 内存可用 |
| Qwen3-8B | 8B | v1.2.1+ 支持 | W4A16 | ~5,000-6,000 | **不测试/OOM** | 8GB 上 W8A8 会 OOM，已排除 |
| Qwen3-14B | 14B | v1.2.1+ 支持 | W4A16 | ~8,000+ | **不可行** | 即使 INT4 也占满内存，加上系统开销必然 OOM |
| Qwen3-32B | 32B | v1.2.1+ 支持 | - | ~16,000+ | **不可行** | 内存严重不足 |

### 1.3 Qwen3-VL 多模态系列

多模态模型除 LLM 解码器外，还需加载 Vision Encoder (RKNN 格式, FP16)，额外占用约 800-900MB。

| 模型 | 参数量 | RKLLM 支持 | 量化方式 | 预估 RAM (MB) | 可行性 | 说明 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| Qwen3-VL-2B | 2B | v1.2.3 支持 | W8A8 | ~1,892 | **可以跑** | 官方 benchmark 数据 |
| Qwen3-VL-4B | 4B | v1.2.3 支持 | W8A8 | ~8,700 | **极限/可能OOM** | Qengineering 实测需 8.7GB |
| Qwen3-VL-8B | 8B | 理论支持 | W8A8 | >10,000 | **不可行** | LLM ~8.8GB + Vision ~900MB |
| Qwen3-VL-32B | 32B | 理论支持 | - | >20,000 | **不可行** | 完全不可行 |
| Qwen3-VL-30B-A3B | 30B (MoE) | 理论支持 | - | >16,000 | **不可行** | MoE 需加载完整 30B 权重 |

## 2. 可行性汇总

### 可以跑 (稳定)

| # | 模型 | 类型 | RAM |
| :--- | :--- | :--- | :--- |
| 1 | InternVL3.5-1B | VLM | ~1.9GB |
| 2 | InternVL3.5-2B | VLM | ~3.0GB |
| 3 | InternVL3.5-4B | VLM | ~5.4GB |
| 4 | Qwen3-0.6B | LLM | ~0.8GB |
| 5 | Qwen3-1.7B | LLM | ~1.7GB |
| 6 | Qwen3-4B | LLM | ~4.3GB |
| 7 | Qwen3-VL-2B | VLM | ~1.9GB |

### 极限测试 (可能 OOM) — 不含 8B

| # | 模型 | 类型 | RAM | 风险 |
| :--- | :--- | :--- | :--- | :--- |
| 1 | Qwen3-VL-4B | VLM | ~8.7GB | 超过物理内存 |

**已排除（不测试）**：Qwen3-8B、InternVL3.5-8B 在 8GB 板子上会 OOM，本项目不测。

### 不可行

| # | 模型 | 原因 |
| :--- | :--- | :--- |
| 1 | Qwen3-14B | W4A16 仍需 ~8GB 纯净，加系统开销 OOM |
| 2 | Qwen3-32B | 需 ~16GB+ |
| 3 | Qwen3-VL-8B | LLM + Vision 合计 >10GB |
| 4 | Qwen3-VL-32B | 需 >20GB |
| 5 | Qwen3-VL-30B-A3B | MoE 需全量权重 >16GB |
| 6 | InternVL3.5-14B | 需 >14GB |
| 7 | InternVL3.5-30B-A3B | MoE 需全量权重 >16GB |
